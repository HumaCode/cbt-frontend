# 📋 Master Plan & Arsitektur Teknis: Aplikasi CBT Universal

## 🏗️ 1. Arsitektur Kode (Clean Architecture)
Aplikasi ini menggunakan pola desain *Clean Architecture* pada *backend* Laravel untuk memisahkan logika bisnis, akses data, dan penyajian API secara terstruktur.
* **Form Request:** Menangani validasi *input* data masuk (contoh: `StoreQuestionRequest`, `RegisterUserRequest`) agar *controller* bersih dari logika validasi.
* **Controller:** Lapis terluar yang hanya bertugas menerima *request* HTTP, memanggil *Service* yang relevan, dan mengembalikan *API Resource*.
* **Service (Business Logic):** Menangani logika bisnis yang kompleks, seperti algoritma perakitan paket soal acak, kalkulasi nilai (IRT/*Auto-grading*), dan penerbitan *token* sesi ujian.
* **Interface & Repository:** Mengabstraksi kueri *database* (Eloquent ORM). *Service* tidak berinteraksi langsung dengan Model, melainkan melalui *Interface* agar kode lebih modular, mudah diuji (*testable*), dan adaptif terhadap perubahan.
* **API Resource:** Memformat respons JSON yang dikembalikan ke *frontend* secara konsisten, aman, dan rapi.

## 🔐 2. Modul Autentikasi (JWT Auth)
Sistem gerbang masuk dan pengelolaan sesi menggunakan standar JSON Web Token (JWT), ideal untuk arsitektur API-*first*.
* **Login & Auth:** Autentikasi kredensial pengguna dan mengembalikan `access_token` JWT beserta *payload* profil pengguna.
* **Register:** Pendaftaran akun baru (khususnya untuk Peserta eksternal) dengan sistem verifikasi data.
* **Token Management:** Fitur *refresh token* untuk memperpanjang sesi tanpa perlu *login* ulang, serta *logout* untuk meng-inaktivasi token secara aman (*blacklist*).

## 🏢 3. Modul Manajemen Pengguna & Otorisasi
Diperkuat oleh *package* **`spatie/laravel-permission`** untuk kontrol akses berbasis peran (RBAC) yang solid dan dinamis.
* **Role & Permission Engine:** Penentuan hak akses granular. 
  * *Contoh Role:* Super Admin, HR Manager, Assessor, Peserta. 
  * *Contoh Permission:* `create-questions`, `grade-essays`, `view-live-leaderboard`.
* **Manajemen Entitas Fleksibel:** Pengelompokan peserta ke dalam *Group* atau *Batch* untuk keperluan rekrutmen atau ujian massal.
* **Mass Data Management:** Fitur *import/export* data peserta massal menggunakan format CSV/Excel.

## 🗂️ 4. Modul Bank Soal & Manajemen Media
Integrasi **`spatie/laravel-medialibrary`** digunakan untuk mengelola lampiran berkas secara elegan, mengaitkan *file* media langsung ke entitas Model `Question` tanpa perlu tabel *database* tambahan yang rumit.
* **Media Management:** Penanganan *upload*, penyimpanan, dan distribusi gambar/audio/dokumen untuk lampiran soal.
* **Kategori & Tagging:** Pengelompokan direktori soal yang terstruktur (TPA, TIU, TKP, Koding, Psikotes).
* **Tipe Pertanyaan Dinamis:** Pilihan Ganda (Single/Multiple Answer), Essay (*Rich Text*), Skala Likert, dan Upload Portofolio.
* **Leveling Soal:** Penandaan tingkat kesulitan (Mudah, Sedang, Sulit) untuk keperluan pengacakan proporsional.

## ⚙️ 5. Modul Konfigurasi Asesmen
* **Perakitan Paket (Static & Dynamic):** Pembuatan set ujian secara manual berdasarkan nomor spesifik, atau diacak otomatis oleh algoritma sistem.
* **Manajemen Waktu Berlapis:** Pengaturan waktu global (total durasi tes) dan waktu parsial (batas pengerjaan per bagian/sesi).
* **Jadwal & Kuota:** Konfigurasi *Start/End Date*, kuota maksimal pendaftar, dan batas percobaan (*Max Attempts*).
* **Token Ujian Dinamis:** PIN atau *passcode* ujian yang di-*generate* otomatis dan berubah secara berkala.

## 💻 6. Modul Eksekusi Tes & Keamanan (Test Engine)
* **Dashboard Peserta:** Portal ujian tunggal yang intuitif.
* **UI Lembar Ujian Terisolasi:** Grid navigasi soal interaktif, *timer* yang dihitung dari *server*, dan *Auto-Save State* via API per perpindahan nomor untuk mencegah hilang jawaban saat perangkat mati.
* **Pengawasan (Proctoring):** Pelacakan aktivitas *tab browser* (*Tab Focus Loss*), pencegahan klik kanan, dan *anti-copy-paste*.

## 🏆 7. Modul Live Monitoring & Gamifikasi
* **Live Leaderboard:** Papan peringkat klasemen peserta secara instan.
* **Live Activity Feed:** Log pengawasan *real-time* untuk memantau peserta yang sedang aktif, selesai, atau kehilangan koneksi.

## 📊 8. Modul Penilaian, Pelaporan & Sertifikasi
* **Engine Penilaian:** Penilaian otomatis (*auto-grading*) soal objektif, kalkulasi Item Response Theory (IRT) untuk pembobotan dinamis, dan antarmuka *Rubric Grading* untuk asesmen manual *essay*.
* **Analitik & Ekspor:** Laporan skor individual, statistik tingkat keberhasilan butir soal, dan fungsi ekspor ke PDF/Excel.
* **Sertifikasi Digital:** Penerbitan *e-sertifikat* otomatis bagi peserta yang lulus *Passing Grade*, dirancang agar siap diintegrasikan dengan layanan segel elektronik (e-meterai) untuk validitas hukum.

---

*Catatan Implementasi Pengembangan:*
Menerapkan pola *Clean Architecture* (Service-Repository-Interface) menuntut pembuatan banyak struktur *file* *boilerplate*. Penggunaan ekstensi AI seperti GitHub Copilot di *code editor* akan sangat memangkas repetisi pengerjaan kerangka dasar ini. Selain itu, untuk menopang performa fitur **Live Leaderboard** tanpa memberatkan REST API utama, direkomendasikan mendelegasikan arus data *real-time* tersebut ke ekosistem terpisah seperti Firebase Studio.
